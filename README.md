# Business Ecosystem Minigame
 
A small browser game that turns a **business ecosystem into a knowledge‑graph puzzle**: connect the right companies before the timer runs out. It was built as a **research artifact** — a case study in using a large language model (LLM) as a *co‑developer*.
 
> Status: playable demo and research prototype — not a finished product. Content is hand‑seeded and static. One experimental "self‑updating" feature was attempted but does **not** work reliably and is not part of the shipped game (see *How it was built*).
 
![Gameplay](docs/gameplay.png)
 
## The idea
 
Business ecosystems — the web of partners, competitors, suppliers and complementors around a firm — are hard to read from static charts. This game makes that network *playable*: nodes are companies or actors, and you score by recovering the valid ties between them under a time limit.
 
It also doubles as a study of LLM‑assisted development: most of the code and game content was generated by an LLM under human direction, with the author(s) setting the context and constraints and then reviewing, correcting, and merging each step.
 
## Play
 
- **Live:** https://tonitani.github.io/GameProject
- Move the cursor to connect the highlighted core companies. Correct ties turn green and raise your score; finish before the timer runs out. Later levels add more nodes, less time, and more required links.
## Run locally
 
```bash
npm install
npm run dev       # local dev server (Vite)
npm run build     # production build
npm run preview   # preview the production build
```
 
Requires Node.js and a modern browser.
 
## What's in it
 
- Multiple levels with different company sets and a short framing story per level.
- Score, high score (saved in the browser via `localStorage`), and a per‑level timer.
- A main menu, basic UI readouts, and simple sound.
- A knowledge‑graph play area where nodes are actors and edges are the ties to recover.
## Tech
 
Phaser 3 (HTML5 game framework) · JavaScript · Vite · deployed on GitHub Pages · state and scores persisted in the browser.
 
## Project structure (high level)
 
- **Core:** `Graph`, `GraphNode`, `GraphEdge`, `Player`, `Timer`, `ScoreManager`
- **Scenes:** `Menu`, `Preload`, `GraphPuzzle`, `LevelComplete`, `GameOver`
- **Level data:** nodes, time limits, layouts, and story/hint text
- **`AGENTS.md`:** standing instructions for any LLM coding agent working in this repo (architecture, constraints, definition‑of‑done, known failure modes)
## How it was built
 
The LLM acted as a co‑developer: it proposed code and game content (level themes, node labels, hint text), and the author set the context and constraints, then checked, corrected, and merged each increment — keeping a change only when the build still ran. Disciplined prompt and context engineering is what made this human‑in‑the‑loop loop reliable.
 
A further step — having an agent **autonomously regenerate content** (a daily randomisation of node labels) — was attempted and **fell short.** Under context drift the agent produced plausible‑but‑wrong code that diverged from the codebase and could not self‑correct.
 
## Research context
 
This repository accompanies a Design Science Research study on LLM‑assisted development. The paper explores a nascent idea — *generative Design Science Research* (geDSR): artifacts that are co‑produced, and only partly regenerated, by a generative agent under human‑in‑the‑loop orchestration. The geDSR idea builds on Verma et al. (2026), *Toward an artifact that designs itself*; here it is adopted in a lighter, human‑in‑the‑loop form rather than a fully autonomous one.
 
 
## Limitations
- Built largely with an older, small‑context model; newer agents may behave differently.
- The evidence on autonomous regeneration rests on one small feature.
